tip for the interview: If an interviewer ever asks how to dynamically check which instances are free in a specific account/region, you can mention the AWS CLI command the error suggested: aws ec2 describe-instance-types --filters Name=free-tier-eligible,Values=true --query "InstanceTypes[*].InstanceType")


This command is a powerful tool for a DevOps engineer because it removes the guesswork when trying to stay within the AWS Free Tier.

When you encountered the "400 Bad Request" error earlier, it was because the t2.micro instance type wasn't available for the Free Tier in your specific account or region. This command allows you to ask the AWS API directly: "What can I run for free right now?"

Here is the breakdown of what each part of that command does:

1. The Command Breakdown
aws ec2 describe-instance-types: This is the base command. Without filters, it would return a massive list of hundreds of different instance types (from tiny t3.nano to massive p4d.24xlarge).

--filters Name=free-tier-eligible,Values=true: This acts like a search filter. It tells AWS to only show instances where the free-tier-eligible attribute is set to true.

--query "InstanceTypes[*].InstanceType": By default, AWS returns data in a long JSON format with hundreds of lines for each instance (showing CPU, memory, network performance, etc.). This "query" part tells the CLI: "I only want the names of the instances. Ignore everything else."

2. Why this is an "Interview Win"
In a DevOps interview, if you are asked about cost optimization or managing AWS resources, mentioning this command demonstrates three things:

CLI Proficiency: You know how to use advanced CLI features like --filters and --query (JMESPath).

Cost Consciousness: You understand the importance of staying within budget or Free Tier limits.

Regional Awareness: You recognize that AWS is not "one size fits all"—available resources and pricing change based on the Region (e.g., US-East-1 vs. US-West-2).

3. How to use it in your terminal
You can run this right now on your MacBook Air to see what else you could have used for Scenario 03:

Bash
aws ec2 describe-instance-types --filters Name=free-tier-eligible,Values=true --query "InstanceTypes[*].InstanceType" --region us-east-1
The Output will look like this:

JSON
[
    "t2.micro",
    "t3.micro"
]
(Note: If you only see t3.micro, it means your account/region combination has moved away from the older t2 family for the Free Tier.)
